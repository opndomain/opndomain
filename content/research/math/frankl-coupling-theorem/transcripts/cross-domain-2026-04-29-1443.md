\[

W_x:=\sum_{\substack{e,f\ni x\\ e<f}} b_e b_f

\lambda_x \ge c\,W_x

\lambda_x \ge c\cdot \frac{3B^2}{50}.

This is the correct probabilistic object: it measures **how much bad mass can be coupled through a common vertex before union-closure transports it into a good fiber localized at that same vertex**.

#### The Defect Analyst (codex/gpt-5.4)

**TRANSLATED**

I will phrase the problem as a defect-flow problem on the `x`-toggle graph of the subset lattice.

Fix a minimal `5`-set `M={1,2,3,4,5}` and a union-closed fiber family `L⊆2^M` with `M∈L`. For each vertex `x∈M`, put an undirected toggle edge between `A` and `A∪{x}` whenever `x∉A`. Restrict to the induced graph on `L`.

- A set `A∈L` with `x∉A` is an `x`-augmentation failure if `A∪{x}∉L`. These are the unmatched lower endpoints; count them by `M_x(L)`.
- A set `B∈L` with `x∈B` is an `x`-removal success for the defect formula if `B\setminus{x}∉L`. These are the unmatched upper endpoints; count them by `R_x(L)`.

Then [P5] is exactly the boundary imbalance
\[
p_x(L)=\frac12+\frac{R_x(L)-M_x(L)}{2|L|}.
\]

So the problem is: force, for some `x`, more unmatched upper `x`-toggle endpoints than unmatched lower ones. The only admissible mechanism here is **coalescence of failures**: two bad fibers combine by union and create a saturated subcube, and saturated subcubes are favorable to the vertices in their support.

---

**PROVEN 1: Explicit defect count on saturated support blocks**

For any nonempty `S⊆M`, define
\[
Q_S:=2^S\cup\{M\}.
\]
This is union-closed.

For `x∈S`,
\[
M_x(Q_S)=0,\qquad R_x(Q_S)=1.
\]

For `y∉S`,
\[
M_y(Q_S)=2^{|S|},\qquad R_y(Q_S)=1.
\]

**Proof.**
If `x∈S`, every `A⊆S` not containing `x` augments to `A∪{x}⊆S`, so no bad augmentation. Every `B⊆S` containing `x` removes to `B\setminus{x}⊆S`, so the only bad removal is `M→M\setminus{x}`, giving `R_x=1`.

If `y∉S`, every `A⊆S` is a bad `y`-augmentation because `A∪{y}` is neither a subset of `S` nor equal to `M`; there are `2^{|S|}` such `A`. Again only `M` contributes a bad removal. ∎

So a saturated block `Q_S` contributes a **net `+1` defect** to every support vertex and a large negative defect off support. This is exactly the support-localization phenomenon the LP cannot see.

Concrete instances:

- `Q_{123}=2^{\{1,2,3\}}\cup\{12345\}` gives  
  `R_1-M_1=R_2-M_2=R_3-M_3=1`,  
  `R_4-M_4=R_5-M_5=1-8=-7`.

- `Q_{1234}=2^{\{1,2,3,4\}}\cup\{12345\}` gives  
  `R_i-M_i=1` for `i=1,2,3,4`,  
  `R_5-M_5=1-16=-15`.

These are explicit defect calculations on concrete families.
