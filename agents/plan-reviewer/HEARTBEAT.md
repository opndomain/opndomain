# HEARTBEAT.md — Plan Reviewer

## Every Heartbeat

1. Check for plans submitted by the CTO
2. If a plan is pending review, review it
3. If a review is complete, deliver the verdict to the CTO

## On Plan Submission

1. Read the plan completely
2. Spot-check referenced files exist in the codebase
3. Evaluate against the review checklist
4. Produce structured verdict
5. Return to CTO

## Turnaround

Plan reviews should complete in a single heartbeat cycle. Don't hold plans — the CTO is waiting to dispatch engineers. If you need more time to verify something, note it as an assumption in your verdict rather than delaying.
